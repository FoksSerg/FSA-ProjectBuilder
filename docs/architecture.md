# Архитектура FSA-ProjectBuilder

## Общая архитектура

FSA-ProjectBuilder построен на модульной архитектуре, которая позволяет легко расширять функциональность.

## Основные компоненты

### Core (Ядро системы)
- **parser.py** - Парсинг Python кода с помощью AST
- **builder.py** - Сборка модулей в один файл
- **rebuilder.py** - Разборка файла на модули
- **validator.py** - Валидация кода
- **comparator.py** - Сравнение файлов
- **dependency_resolver.py** - Разрешение зависимостей

### GUI (Графический интерфейс)
- **main_window.py** - Главное окно приложения
- **tree_view.py** - Дерево проекта
- **graph_view.py** - Граф зависимостей
- **diff_view.py** - Сравнение файлов
- **process_view.py** - Окно процесса
- **settings_dialog.py** - Настройки

### Distributors (Создание дистрибутивов)
- **base.py** - Базовый класс дистрибутора
- **pyinstaller.py** - PyInstaller интеграция
- **cx_freeze.py** - cx_Freeze интеграция
- **nuitka.py** - Nuitka интеграция

### Utils (Утилиты)
- **ast_utils.py** - Утилиты для работы с AST
- **file_utils.py** - Работа с файлами
- **config_utils.py** - Работа с конфигурацией
- **code_formatter.py** - Форматирование кода
- **code_cleaner.py** - Очистка кода

## Поток данных

### Разборка (Rebuild)
1. Чтение исходного файла
2. Парсинг AST
3. Определение структуры
4. Создание модулей
5. Генерация __init__.py
6. Сохранение метаданных

### Сборка (Build)
1. Чтение метаданных
2. Загрузка модулей
3. Разрешение зависимостей
4. Объединение кода
5. Применение настроек
6. Создание файла
7. Валидация

### Создание дистрибутива (Dist)
1. Сборка проекта
2. Определение зависимостей
3. Выбор инструмента
4. Создание дистрибутива
5. Валидация результата
